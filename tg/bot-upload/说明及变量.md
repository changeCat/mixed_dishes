# Telegram Bot 图床上传工具

## 功能概述

通过 Telegram Bot 实现文件上传到 Sanyue ImgHub 图床，支持多渠道、多目录上传。

## 核心特性

### 1. 文件上传
- **单文件上传**：发送单个文件自动显示上传面板
- **批量上传**：相册自动识别，支持统一上传或分别上传
- **外链上传**：支持识别消息中的 URL 链接并上传
- **无 KV 兜底**：未绑定 KV 时使用内存临时缓存

### 2. 命令功能

#### 私聊命令（Private Chat）
- `/list` - 📂 浏览图床目录，支持分页查看文件信息
- `/random` - 🎲 随机图片/视频漫游，支持目录切换
- `/reset` - 🔄 重置上传缓存（清空 KV 临时数据）
- `/init` - ⚙️ 强制刷新命令菜单

#### 群组/频道命令（Group/Channel）
- `/info` - ℹ️ 查看消息元数据（支持回复消息查看）
- `/delete` - 🗑 删除图床文件（需回复目标图片消息，二次确认）

### 3. 交互优化
- **渠道动态切换**：支持 Radio Button 风格的渠道选择（✅ 已选 / ⬜ 未选）
- **秒回体验**：批量上传立即显示"加载中"提示，优化等待时间
- **防重复触发**：按钮点击后立即禁用，防止重复提交
- **自动清理**：命令消息和提示消息自动延迟删除（7秒后）

### 4. 数据存储

需要创建并绑定两个 KV 数据库：
- **TG_KV**：Bot Workers 内部临时缓存（变量名：`TG_KV`）
- **img_url**：图床索引数据（用于 `/delete` 命令匹配文件）

## 环境变量配置

| 配置参数 | 说明 | 必填 |
|---|---|---|
| **TG_BOT_TOKEN** | Telegram Bot Token | ✅ |
| **TG_BOT_SECRET** | Webhook 验证密钥，配置方法：<br>`https://api.telegram.org/bot<TOKEN>/setWebhook?url=<WORKER_URL>&secret_token=<SECRET>` | ✅ |
| **ALLOWED_USERS** | 允许使用的 Telegram 用户 ID，逗号分隔（如：`123456,789012`） | ✅ |
| **API_UPLOAD_URL** | 图床上传 API 地址（如：`https://xxx.pages.dev/upload`） | ✅ |
| **API_UPLOAD_TOKEN** | 图床上传权限 Token | ✅ |
| **API_LIST_TOKEN** | 图床列表查询权限 Token（用于 `/list` 命令） | ⚠️ |
| **API_DELETE_TOKEN** | 图床删除权限 Token（用于 `/delete` 命令） | ⚠️ |
| **ACCESS_URL** | 图床外链访问域名（自定义域名，如：`https://img.example.com`） | ⚠️ |
| **DIR_LIST** | 上传目录列表，逗号分隔（如：`default,icon,img,gif`） | ⚠️ |
| **DEFAULT_DIR** | 默认上传目录（默认：`default`） | ❌ |
| **CHANNEL_LIST** | 上传渠道配置，支持子频道，格式：<br>`名称:提供商:子频道名`<br>示例：`TG:telegram,Main:telegram:main,HF:huggingface`<br>默认：`TG:telegram` | ❌ |

### 渠道配置说明（CHANNEL_LIST）

格式：`显示名称:提供商[:子频道名]`

- **基础配置**：`TG:telegram` → 显示为 "TG"，实际渠道为 `telegram`
- **子频道配置**：`Main:telegram:main` → 显示为 "Main"，实际传递 `uploadChannel=telegram&channelName=main`
- **多渠道示例**：
  ```
  TG:telegram,Main:telegram:main,Backup:telegram:backup,HF:huggingface
  ```

## 部署步骤

### 1. Webhook 配置
```bash
# 设置 Webhook
https://api.telegram.org/bot<YOUR_BOT_TOKEN>/setWebhook?url=<YOUR_WORKER_URL>&secret_token=<YOUR_SECRET>

# 验证 Webhook 状态
https://api.telegram.org/bot<YOUR_BOT_TOKEN>/getWebhookInfo
```

### 2. KV 绑定
在 Cloudflare Workers 设置中绑定：
- 变量名：`TG_KV` → 绑定到 Bot 临时缓存 KV
- 变量名：`img_url` → 绑定到图床索引 KV

### 3. 环境变量设置
在 Workers 设置中配置所有必填环境变量。

### 4. 初始化命令
首次部署后，在私聊中发送 `/init` 强制刷新命令菜单。

## 技术细节

### 文件类型支持
- **图片**：jpg, jpeg, png, gif, webp, bmp
- **视频**：mp4, webm, mov, mkv, avi, m4v, flv（支持流式播放）
- **文档**：支持任意文件上传

### 时区处理
所有时间戳自动转换为 UTC+8（北京时间）显示。

### 错误处理
- 批量上传失败时显示详细错误信息
- API 请求超时设置为 8 秒
- 自动清理过期的临时缓存
